## 🏗️ 数据隔离架构概述

### 设计原则
1. **默认隔离**: 每个用户的个人数据完全独立存储
2. **配对共享**: 只有在配对成功后，特定数据才会互通
3. **灵活分离**: 取消配对时可选择数据处理方式
4. **数据安全**: 用户无法访问其他用户的私有数据
5. **向后兼容**: 新架构兼容现有数据结构

### 架构优势
- **隐私保护**: 用户数据完全隔离，保护个人隐私
- **功能灵活**: 支持单用户和情侣模式的无缝切换
- **扩展性强**: 支持任意数量用户的独立数据管理
- **数据完整**: 每个用户都有完整的个人数据副本

## 📊 数据分类体系

### 1. 个人私有数据 (用户隔离)

#### 存储格式
```
AsyncStorage:
├── @account_records_{userId}     // 个人记账记录
├── @mood_records_{userId}        // 个人心情记录
├── @user_settings_{userId}       // 个人设置
├── @account_categories_{userId}  // 个人记账分类
└── @notification_settings_{userId} // 个人通知设置
```

#### 特征
- **完全隔离**: 不同用户之间无法访问
- **独立操作**: 所有CRUD操作仅影响当前用户数据
- **持久保存**: 即使配对/取消配对也不会丢失

#### 数据示例

### 2. 情侣共享数据 (配对时互通)

#### 存储格式
```
AsyncStorage:
├── @couple_info                 // 配对信息
├── @couple_accounts             // 共同记账
├── @couple_transactions         // 共同交易记录
├── @my_messages                 // 我的留言
├── @partner_messages            // 伙伴留言
├── @couple_plans                // 共同计划
├── @couple_settings             // 情侣设置
└── @couple_mood_sync            // 心情同步设置
```

#### 特征
- **配对激活**: 只有在配对状态下才会显示和操作
- **双向同步**: 双方都可以查看和操作（根据权限）
- **权限控制**: 不同功能有不同的权限设置
- **分离处理**: 取消配对时可选择保留或删除

### 3. 全局共享数据

#### 存储格式
```
AsyncStorage:
├── @saved_credentials           // 保存的登录凭据
├── @app_settings               // 应用全局设置
└── @last_login_user            // 最后登录用户
```

#### 特征
- **设备级别**: 在设备上全局共享
- **功能性**: 主要用于应用功能，不涉及用户隐私数据

## 🔧 技术实现

### 1. 存储键生成函数

```typescript
/**
 * 生成用户特定的存储键
 * @param baseKey 基础存储键
 * @param userId 用户ID
 * @returns 用户特定的存储键
 */
const getUserStorageKey = (baseKey: string, userId?: string): string => {
  if (!userId) {
    return baseKey;
  }
  return `${baseKey}_${userId}`;
};

// 使用示例
const currentUser = authService.getCurrentUser();
const storageKey = getUserStorageKey('@account_records', currentUser?.id);
// 结果: @account_records_1 或 @account_records_2
```

### 2. 数据操作模式

#### 修改前 (全局共享模式)
```typescript
// ❌ 错误：所有用户共享同一份数据
static async getAllRecords(): Promise<AccountRecord[]> {
  const records = await AsyncStorage.getItem('@account_records');
  return records ? JSON.parse(records) : [];
}
```

#### 修改后 (用户隔离模式)
```typescript
// ✅ 正确：每个用户有独立的数据
static async getAllRecords(): Promise<AccountRecord[]> {
  const { authService } = await import('./AuthService');
  const currentUser = authService.getCurrentUser();
  const storageKey = getUserStorageKey('@account_records', currentUser?.id);
  
  const records = await AsyncStorage.getItem(storageKey);
  return records ? JSON.parse(records) : [];
}
```

#### 取消配对数据处理
```typescript
const handleDisconnect = async (option: 'keep_all' | 'keep_mine' | 'delete_all') => {
  switch (option) {
    case 'keep_all':
      // 保留个人数据 + 共享数据转为个人回忆
      await convertSharedToPersonal();
      break;
    case 'keep_mine': 
      // 保留个人数据 + 删除伙伴相关数据
      await deletePartnerData();
      break;
    case 'delete_all':
      // 只保留个人数据 + 删除所有共享数据
      await clearAllSharedData();
      break;
  }
};
```

### 2. 代码规范示例

#### ✅ 正确的个人数据操作
```typescript
// 个人数据操作 - 使用用户隔离
static async createPersonalRecord(data: RecordData): Promise<Record> {
  const { authService } = await import('./AuthService');
  const currentUser = authService.getCurrentUser();
  const storageKey = getUserStorageKey('@records', currentUser?.id);
  
  const records = await AsyncStorage.getItem(storageKey);
  const updatedRecords = [...(records ? JSON.parse(records) : []), newRecord];
  await AsyncStorage.setItem(storageKey, JSON.stringify(updatedRecords));
  
  return newRecord;
}
```

#### ✅ 正确的共享数据操作
```typescript
// 共享数据操作 - 使用固定键
static async createSharedRecord(data: SharedData): Promise<SharedRecord> {
  const storageKey = '@couple_shared_records'; // 固定键，不需要用户ID
  
  const records = await AsyncStorage.getItem(storageKey);
  const updatedRecords = [...(records ? JSON.parse(records) : []), newRecord];
  await AsyncStorage.setItem(storageKey, JSON.stringify(updatedRecords));
  
  return newRecord;
}
```

#### ❌ 错误的数据操作
```typescript
// 错误：个人数据使用全局键
static async createRecord(data: RecordData): Promise<Record> {
  const storageKey = '@global_records'; // ❌ 个人数据不应该全局共享
  // ...
}
```

## 🚨 重要注意事项

### 1. 数据安全
- **绝对隔离**: 个人数据必须完全隔离，不能有任何泄露
- **权限控制**: 共享数据要有适当的权限控制
- **数据完整性**: 任何操作都不能导致数据丢失

### 2. 性能考虑
- **存储效率**: 避免重复存储相同数据
- **查询优化**: 合理设计数据结构提高查询效率
- **内存管理**: 及时释放不需要的数据

### 3. 向后兼容
- **数据迁移**: 新架构要能处理旧版本数据
- **渐进升级**: 可以逐步迁移不同类型的数据
- **回滚支持**: 必要时能够回滚到旧版本
