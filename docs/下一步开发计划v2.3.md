## 🚀 下一阶段开发重点

### 优先级 1: 情侣共同记账功能 ⭐⭐⭐

#### 功能概述
基于已完成的用户数据隔离架构，实现情侣双方共享的记账系统，支持共同资金管理和权限控制。

#### 架构基础 🔴
- **数据隔离**: 个人记账数据保持隔离，共同记账使用独立存储
- **权限控制**: 基于配对状态的数据访问控制
- **数据处理**: 取消配对时的灵活数据处理选项

#### 核心功能
1. **共同金库系统**
   - 双方共享的资金池
   - 存入/取出记录管理
   - 余额实时同步

2. **权限管理系统**
   - 三种权限模式：
     - 总是需要批准
     - 自定义金额阈值
     - 仅报告模式
   - 权限设置界面

3. **申请审批流程**
   - 卡片式申请界面
   - 实时通知对方
   - 批准/拒绝操作
   - 申请历史记录

4. **数据同步机制**
   - 双方记录实时同步
   - 冲突解决策略
   - 离线数据处理

#### 技术实现要点
- 扩展现有 AccountingService
- 新增 CoupleAccountingService
- 实现权限控制逻辑
- 设计申请审批UI组件
