## 🏗️ 数据架构设计

### 数据分类体系

#### 个人私有数据 (始终隔离)
```
@account_records_{userId}     // 个人记账数据
@mood_records_{userId}        // 个人心情数据  
@user_settings_{userId}       // 个人设置
@saved_credentials           // 登录凭据
```

#### 情侣共享数据 (配对时互通)
```
couple_accounts              // 共同记账
couple_transactions          // 共同交易记录
@my_messages                 // 我的留言
@partner_messages            // 伙伴留言
@couple_plans                // 共同计划
@couple_info                 // 配对信息
```

### 配对状态数据处理逻辑

#### 未配对状态
- 只显示个人数据
- 所有操作仅影响个人数据存储

#### 配对状态  
- 显示个人数据 + 共享数据
- 个人操作仍然隔离存储
- 共同功能操作共享数据

#### 取消配对
- 提供三种数据处理选项:
  1. **保留所有回忆**: 保留个人+共享数据
  2. **只保留我的**: 保留个人数据+删除伙伴相关数据  
  3. **删除所有**: 只保留个人数据+删除所有共享数据

## 🛠️ 技术实现细节

### 1. 用户存储键生成函数
```typescript
const getUserStorageKey = (baseKey: string, userId?: string): string => {
  if (!userId) {
    return baseKey;
  }
  return `${baseKey}_${userId}`;
};
```

### 2. 数据操作模式改进
```typescript
// 修改前 (全局共享)
const records = await AsyncStorage.getItem(STORAGE_KEYS.ACCOUNT_RECORDS);

// 修改后 (用户隔离)
const { authService } = await import('./AuthService');
const currentUser = authService.getCurrentUser();
const storageKey = getUserStorageKey(STORAGE_KEYS.ACCOUNT_RECORDS, currentUser?.id);
const records = await AsyncStorage.getItem(storageKey);
```

### 3. 测试数据创建逻辑
```typescript
// 根据用户ID创建不同的测试数据
if (currentUser.id === '1') {
  // test用户的普通上班族数据
} else if (currentUser.id === '2') {
  // test1用户的设计师数据  
} else {
  // 其他用户的默认数据
}
```

## 📈 架构优势

### 1. 数据安全性
- **隐私保护**: 用户数据完全隔离，无法互相访问
- **数据完整性**: 每个用户都有完整的个人数据副本
- **操作安全**: 所有操作都在用户范围内进行

### 2. 功能灵活性

- **情侣模式**: 配对后可以共享特定数据

### 3. 扩展性
- **新功能开发**: 明确的数据分类指导新功能设计
- **用户规模**: 支持任意数量用户的独立数据
- **功能组合**: 个人功能和情侣功能可以灵活组合
